Whitespace стеганографія

Принцип

Використання комбінацій пробілів та табуляцій для кодування бінарних даних

Базове кодування: ─────────────────── Пробіл (space) = 0 Табуляція (tab) = 1 Приклад: приховати "A" (ASCII 65 = 01000001) "Hello[SP][TAB][SP][SP][SP][SP][SP][TAB]World"

Місця вбудовування:

  • Кінець рядків
  • Між словами
  • Між абзацами

Типова ємність:

  • ~3 біти на 8 колонок
  • ~1 біт на рядок

Інструмент SNOW

SNOW — популярний інструмент whitespace стеганографії

# Кодування повідомлення stegsnow -C -m "Secret message" -p "password" input.txt output.txt # Декодування stegsnow -C -p "password" output.txt

Особливості SNOW:

  • Додає послідовності до 7 пробілів з табуляціями в кінці рядків
  • Підтримує ICE шифрування для додаткової безпеки
  • Доступний для Linux/Windows

Trailing whitespace часто видаляється текстовими редакторами!

Маніпуляція закінченнями рядків

Різні ОС використовують різні символи закінчення рядків

Windows

\r\n

CR + LF

Unix/Linux

\n

LF

Classic Mac

\r

CR

Ємність: 2 біти на рядок (4 можливі комбінації)

Git та багато редакторів автоматично нормалізують line endings!

Unicode Zero-Width символи

Unicode містить невидимі символи нульової ширини

Символ Unicode Назва Біт
ZWSP U+200B Zero Width Space 0
ZWNJ U+200C Zero Width Non-Joiner 1
ZWJ U+200D Zero Width Joiner -
WJ U+2060 Word Joiner -

Текст виглядає ідентично — символи повністю невидимі!

Алгоритм Unicode кодування

def encode_message(secret, cover_text): # Конвертуємо в бінарний binary = ''.join(format(ord(c), '08b') for c in secret) # Кодуємо zero-width символами encoded = "" for bit in binary: if bit == '0': encoded += '\u200B' # ZWSP else: encoded += '\u200C' # ZWNJ # Вставляємо в середину cover-тексту mid = len(cover_text) // 2 return cover_text[:mid] + encoded + cover_text[mid:]
Ємність = len(cover_text) × біти_на_позицію

При 3-бітовому кодуванні (8 різних ZW-символів) — вища ємність

HTML/CSS стеганографія

Маніпуляція регістром тегів:

<!-- Оригінал --> <HTML><BODY><P>Text</P></BODY></HTML> <!-- З прихованим "10" --> <html><BODY><P>Text</P></BODY></html> lowercase = 0, uppercase = 1

Перестановка атрибутів:

<!-- Біт 0 --> <img src="#" alt="photo" width="100"> <!-- Біт 1 (атрибути переставлені) --> <img alt="photo" src="#" width="100">

Ємність: 1 біт на тег з 2+ атрибутами

CSS-based приховування

/* Приховані дані в значеннях кольорів */ .class { color: #000001; /* LSB кодує 1 біт */ background: #FFFFFE; /* LSB кодує 0 */ } /* Надлишкові значення */ .element { margin: 10px 10px; /* Може бути margin: 10px */ padding: 5px 5px 5px 5px; /* Може бути padding: 5px */ } /* Порядок властивостей */ .item { color: red; /* Порядок A = біт 0 */ font-size: 12px; } /* vs */ .item { font-size: 12px; /* Порядок B = біт 1 */ color: red; }

CSS нормалізація браузерами може зруйнувати приховані дані

Вразливості форматних методів

Whitespace

  • Текстові редактори видаляють trailing spaces
  • Copy-paste часто нормалізує пробіли
  • Email системи переформатовують текст

Unicode Zero-Width

  • Деякі платформи видаляють ZW-символи
  • Легко виявити аналізом символів
  • Соціальні мережі часто фільтрують

HTML/CSS

  • HTML minifiers руйнують дані
  • Браузери нормалізують атрибути
  • CMS можуть переформатувати код
