Заміна синонімів

Принцип

Заміна слів семантично еквівалентними синонімами для кодування бітів

Приклади пар (англійська):

big / large small / tiny fast / quick happy / glad

Приклади пар (українська):

великий / чималий швидкий / прудкий гарний / вродливий
Ємність: ~0.67 біт на речення

Алгоритм вершинного кодування

Chang & Clark — систематичний підхід до синонімічної стеганографії

Алгоритм:

  1. Побудувати граф синонімів
  2. Слова — вершини
  3. Синоніми — ребра
  4. Розфарбування графа → бінарні коди
  5. Заміна за секретними бітами

Приклад графа:

big ──── large │ │ └── huge ─┘ big=0, large=1, huge=?

Не всі синоніми взаємозамінні у всіх контекстах!

Проблеми синонімічного методу

Контекстна залежність

"швидкий поїзд" ✓
"прудкий поїзд" ?

Колокації

"strong tea" ✓
"powerful tea" ✗

Статистичні зміни

Незвичний розподіл слів може бути виявлений

Обмежена ємність

Не кожне слово має придатний синонім

Потребує NLP інструментів: WordNet, тезауруси, мовні моделі

Синтаксичні трансформації

Маніпуляція структурою речення зі збереженням значення

Актив → Пасив: "Кіт зловив мишу" → "Миша була зловлена котом" (біт 0) (біт 1) Cleft-речення: "Іван з'їв торт" → "Це Іван з'їв торт" (біт 0) (біт 1) Порядок прикметників: "гарний великий будинок" → "великий гарний будинок" (біт 0) (біт 1)
Ємність: 0.5-1.0 біт на речення

Семантичні методи

Найскладніший підхід — маніпуляція значенням на концептуальному рівні

Техніки:

Парафразування

"Він прийшов о 15:00"
→ "Його прибуття було о 15:00"

Переупорядкування

"Спочатку A, потім B"
→ "B після A"

Пресупозиції

"Він знову запізнився"
(implies: запізнювався раніше)

Імплікатури

Непряме вираження думки

Непрактичні з поточними NLP технологіями для надійної автоматизації

RNN-Stega: Нейронні мережі

RNN навчена на текстовому корпусі генерує cover-текст

Процес:

1
Навчити RNN
2
Передбачити P(слово)
3
Біти → вибір
4
Згенерувати
P(wt | w1, ..., wt-1) = RNN(context)

bits_embedded = log2(|candidates|)

Секретні біти керують вибором з кандидатів на основі ймовірностей

Adaptive Dynamic Grouping (ADG)

Алгоритм:

  1. Отримати розподіл ймовірностей P по словнику
  2. Динамічно розділити словник на 2r груп з рівною ймовірністю
  3. Кожна група представляє r-бітовий сегмент повідомлення
  4. Вибрати слово з вибраної групи

Ємність

r бітів на токен

Безпека

Доказово безпечний за KL-дивергенцією

Згенерований текст статистично невідрізнимий від природного!

LLM-based стеганографія (2024)

LLM-Stega — робота з black-box API (GPT, Claude, LLaMA)

Компоненти:

1. Побудова keywords: secret → ключові слова 2. Стеганографічне відображення: keywords → промпт 3. Генерація: LLM(промпт) → cover-текст 4. Витягування: cover-текст → secret Приклад: "Зустріч о 15:00" → ["meeting", "afternoon", "time"] → Prompt: "Write about scheduling a business meeting..." → LLM генерує природний текст з keywords
