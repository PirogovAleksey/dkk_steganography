Що таке стеганографія?

Визначення

Стеганографія (грец. steganos — прихований, graphein — писати) — наука про приховану передачу інформації шляхом збереження в таємниці самого факту передачі.

На відміну від криптографії, де повідомлення зашифроване, але його наявність очевидна, стеганографія ховає сам факт існування секретного повідомлення.

Аналогія

Криптографія — ви бачите сейф, але не знаєте код.

Стеганографія — ви навіть не знаєте, що сейф існує.

Як працює LSB-стеганографія
Оригінал
192 140 96
Після LSB
193 141 97
11000000 → 11000001 Змінюється лише останній біт

Різниця ±1 на канал — абсолютно непомітна людському оку

Що ми шукаємо?

Формат прапора (Flag)

У CTF-змаганнях ваша мета — знайти прапор (flag). Це рядок у спеціальному форматі:

FLAG{приклад_прапора_тут} CTF{інший_формат} flag{малими_літерами}

Як шукати:

  • strings file | grep -i flag — пошук у текстових рядках
  • grep -ri "FLAG{" . — рекурсивний пошук у витягнутих файлах
  • У CyberChef шукай патерн FLAG\{.*\} у результаті декодування
  • Прапор може бути закодований (base64, hex, rot13) — декодуй перед пошуком!

Основні методи стеганографії у CTF

LSB (Least Significant Bit)

Найпопулярніший метод. Дані записуються у найменш значущі біти пікселів зображення. Зміна ±1 не помітна оку.

zsteg stegsolve StegOnline

Метадані (EXIF / PNG)

Інформація ховається у метаданих файлу: EXIF, PNG text chunks, IPTC, XMP. Перевіряй exiftool!

exiftool pngcheck identify

Вбудовані файли

Після кінця PNG можна дописати ZIP. Зображення відкриється нормально, але binwalk знайде вкладення.

binwalk foremost xxd

Аудіо стеганографія

Дані можуть бути у LSB аудіо, у спектрограмі (видно як зображення) або в ехо-сигналі.

Audacity sonic-visualiser sox

Текстова стеганографія

Пробіли/табуляції в кінці рядків кодують біти. Unicode zero-width символи невидимі, але несуть інформацію.

cat -A xxd Python

Ланцюжки кодувань

Прапор закодований кілька разів: base64, ROT13, hex, binary. CyberChef Magic автоматично розпізнає ланцюжок!

CyberChef base64 Python

DCT стеганографія

Дані ховаються у частотних коефіцієнтах JPEG (DCT). Стійкіше за LSB, але складніше для витягнення. Використовується steghide та outguess.

steghide outguess stegdetect

Відео стеганографія

Дані ховаються у окремих кадрах відео через LSB або у метаданих контейнера. Витягни кадри через ffmpeg та аналізуй кожен як зображення.

ffmpeg ffprobe Python

Стегоаналіз: як виявити приховані дані

Що таке стегоаналіз?

Стегоаналіз — це наука про виявлення прихованих повідомлень у файлах. Якщо стеганографія ховає дані, то стегоаналіз намагається їх знайти. У CTF це ключовий навик: потрібно не лише знати інструменти, а й розуміти як і чому вони працюють.

Chi-Square аналіз

Статистичний тест, що порівнює розподіл значень LSB з очікуваним рівномірним. Якщо розподіл занадто ідеальний — є приховані дані.

stegdetect Python/scipy

RS-аналіз

Метод Regular-Singular груп. Аналізує співвідношення «гладких» та «шумних» груп пікселів. Дозволяє оцінити відсоток заповнення LSB.

Aletheia Python

Візуальна атака

Перегляд окремих bit planes зображення. У чистому зображенні LSB-площина виглядає як шум; при наявності стего — з’являються паттерни.

StegOnline Stegsolve

Метрики якості

Порівняння з оригіналом (якщо доступний): PSNR, SSIM, MSE. Якщо якість підозріло знижена — можливо, є вбудовані дані.

Python/PIL GIMP ImageMagick

Практичні поради зі стегоаналізу

  • Завжди починай з візуальної атаки — завантаж зображення у StegOnline та переглянь bit planes (особливо біт 0 кожного каналу)
  • Порівнюй з оригіналом — якщо є чисте зображення, вирахуй різницю пікселів: ImageChops.difference(img1, img2)
  • Перевіряй гістограму — стего-зображення часто мають характерні «гребінці» (pairs of values) на гістограмі
  • Стежи за розміром файлу — якщо PNG або BMP файл непропорційно великий для свого розміру — в ньому можуть бути дані

Алгоритм розв'язання Stego CTF

Крок за кроком

1 file / xxd | head — визначити реальний тип файлу
↓
2 strings / exiftool — шукати прапор у тексті та метаданих
↓
3 binwalk — перевірити на вбудовані файли
↓
4 Зображення? → zsteg / StegOnline / Aperi'Solve
Аудіо? → спектрограма + audio LSB

↓
5 CyberChef Magic — автоматичне декодування ланцюжка
↓
6 Якщо зашифровано — шукати пароль в інших шарах
↓
7 FLAG{...}

Покроковий приклад: розв'язуємо challenge

Сценарій

Маємо файл mystery.png (звичайне фото котика). Треба знайти прапор.

1

Визначаємо тип і шукаємо очевидне

$ file mystery.png mystery.png: PNG image data, 800 x 600, 8-bit/color RGBA $ strings mystery.png | grep -i flag (нічого не знайдено)

Файл справді PNG. Прапор не лежить відкритим текстом — йдемо далі.

2

Перевіряємо метадані

$ exiftool mystery.png ... Comment: aHR0cHM6Ly9leGFtcGxlLmNvbQ== Author: Nothing here ...

Є підозрілий коментар! Виглядає як base64 (літери A-Z, a-z, цифри, закінчується на ==).

3

Декодуємо знахідку

$ echo "aHR0cHM6Ly9leGFtcGxlLmNvbQ==" | base64 -d https://example.com

Це URL, але не прапор. Можливо це підказка, а прапор в іншому шарі?

4

Перевіряємо вбудовані файли та LSB

$ binwalk mystery.png DECIMAL HEXADECIMAL DESCRIPTION 0 0x0 PNG image, 800 x 600 (нічого додаткового) $ zsteg mystery.png b1,rgb,lsb,xy .. text: "FLAG{st3g0_m4st3r_2026}"

zsteg знайшов прапор у LSB! FLAG{st3g0_m4st3r_2026}

Підсумок

Алгоритм спрацював: file → strings → exiftool (знайшли base64-підказку) → binwalk → zsteg (знайшли прапор). Весь процес зайняв ~2 хвилини.

Підготовка операційної системи

Рекомендовано

Kali Linux

Всі інструменти вже встановлені: exiftool, binwalk, steghide, foremost, strings, xxd. Просто завантаж та працюй.

sudo apt update && sudo apt upgrade # Все вже є! Додатково: gem install zsteg pip install stegoveritas

Ubuntu / Debian

Потрібно встановити інструменти вручну, але все доступно через apt та pip.

sudo apt install exiftool binwalk steghide foremost pip install stegoveritas gem install zsteg

Windows (WSL)

Встанови WSL2 з Ubuntu, далі як на Linux. Або використовуй онлайн-інструменти.

wsl --install -d Ubuntu # Далі в WSL: sudo apt install exiftool binwalk steghide

macOS

Використовуй Homebrew для встановлення CLI-інструментів.

brew install exiftool binwalk foremost pip install stegoveritas gem install zsteg
Швидкий старт

Docker stego-toolkit

Все в одному контейнері: 20+ CLI-інструментів + GUI (Stegsolve, SonicVisualiser, DeepSound). Одна команда — і готово.

docker pull dominicbreuker/stego-toolkit docker run -it --rm -v $(pwd)/data:/data \ dominicbreuker/stego-toolkit /bin/bash # Автоматичний аналіз: check_png.sh image.png check_jpg.sh image.jpg

Немає часу встановлювати?

Використовуй онлайн-інструменти: CyberChef, StegOnline, Aperi'Solve та HexEd.it. Вони покривають 80% типових CTF-завдань зі стеганографії.

Командний рядок (CLI)

must have

exiftool

Читає ВСІ метадані: EXIF, PNG text, IPTC, XMP. Перший крок для будь-якого файлу.

apt install libimage-exiftool-perl

binwalk

Сканує файл на вбудовані архіви, зображення та інші файли. -e для автоматичного витягнення.

pip install binwalk

zsteg

Автоматичний LSB-аналіз PNG/BMP. -a перебирає всі варіанти бітів та каналів.

gem install zsteg (Ruby) або StegOnline онлайн

steghide

Стеганографія в JPEG та WAV з паролем. steghide extract -sf file.jpg

strings

Витягує всі текстові рядки з бінарного файлу. strings file | grep FLAG

вбудована в Linux/macOS

foremost

Альтернатива binwalk для відновлення файлів за сигнатурами (magic bytes).

apt install foremost

stegseek

Блискавичний брутфорс паролів для steghide. Перебирає rockyou.txt за секунди.

apt install stegseek

pngcheck

Перевіряє структуру PNG: пошкоджені chunks, CRC-помилки, tEXt-дані. -vtp7f для повного аналізу.

apt install pngcheck
must have

ffmpeg / ffprobe

Витягнення кадрів з відео, конвертація аудіо, аналіз метаданих контейнера. ffmpeg -i video.mp4 frame_%04d.png

apt install ffmpeg

Словники паролів (Wordlists)

Навіщо потрібні словники?

Багато CTF-завдань використовують steghide з паролем. Інструменти stegseek та fcrackzip перебирають паролі зі словника. Найпопулярніший — rockyou.txt (14 млн паролів).

# Kali Linux — вже є: /usr/share/wordlists/rockyou.txt.gz sudo gunzip /usr/share/wordlists/rockyou.txt.gz # Ubuntu / інші — завантажити: wget https://github.com/brannondorsey/naive-hashcat/releases/download/data/rockyou.txt # Використання: stegseek mystery.jpg rockyou.txt fcrackzip -u -D -p rockyou.txt archive.zip john --wordlist=rockyou.txt hash.txt

Зламані magic bytes

Перші байти файлу підмінені або обнулені. Відкрий у hex-редакторі та встанови правильну сигнатуру (див. таблицю вище).

Розміри зображення змінені — частина картинки обрізана. pngcheck знайде помилку, а виправлення CRC покаже приховану частину.

Змінене розширення

file xxd binwalk

Часті помилки початківців

  • Пропускають strings — іноді прапор лежить відкритим текстом!
  • Не перевіряють метадані — exiftool має бути першим кроком
  • Забувають про binwalk — багато файлів містять вкладення
  • Не знають про CyberChef Magic — він автоматично розпізнає кодування
  • Використовують тільки 1-бітну LSB — бувають 2-бітні та 4-бітні варіанти
  • Не пробують порожній пароль у steghide — іноді пароль просто пустий!
  • Не перевіряють pngcheck — пошкоджена структура може ховати дані

PSNR / SSIM

python -c " import numpy as np; from PIL import Image a = np.array(Image.open('orig.png'), float) b = np.array(Image.open('steg.png'), float) mse = np.mean((a - b) ** 2) psnr = 10 * np.log10(255**2 / mse) if mse > 0 else float('inf') print(f'MSE={mse:.2f}, PSNR={psnr:.2f} dB') "
Кодування: base64, hex, rot13, binary

Розпізнавання

  • Base64: A-Z, a-z, 0-9, +, / з = на кінці
  • Hex: тільки 0-9, a-f, парна кількість
  • Binary: тільки 0 і 1, групи по 8

Декодування

echo "SGVsbG8=" | base64 -d # base64 echo "48656c6c6f" | xxd -r -p # hex echo "Uryyb" | tr 'A-Za-z' 'N-ZA-Mn-za-m' # rot13
